Visão Geral

LuaSQL é uma interface simples entre Lua e um DBMS. Ela permite que um programa Lua:

  • Conecte-se aos bancos de dados ODBC, ADO, Oracle, MySQL, SQLite, JDBC e PostgreSQL;
  • Execute comandos arbritários do SQL;
  • Recupere resultados no modo linha-a-linha de um cursor SQL.

LuaSQL é um software livre e utiliza a mesma licença do Lua 5.0.

Status

A versão LuaSQL 2.0.0 (para Lua 5.0) está disponível para download.

O driver PostgreSQL foi testado em Linux e MacOS X e é compatível com PostgreSQL 7.x. O driver ODBC foi testado em Windows (drivers SQLServer e Microsoft Access). O driver MySQL foi testado em Linux e é compatível com as versões 4.0 e 4.1. O driver Oracle foi testado em Windows e é compatível com OCI 8 API. O driver SQLite foi testado em Linux Linux e é compatível com as versões 2.x. O driver JDBC foi testado com LuaJava Beta 3 e JDK 1.4 (driver MySQL). O driver ADO foi testado em Windows, com LuaCOM 1.3 (driver Microsoft Access).

Download

O download do código fonte do LuaSQL pode ser feito a partir da sua página no LuaForge.

História

  • [22/03/2005] Lançamento da Versão 2.0.0
  • [23/12/2004] Lançamento da Versão 2.0 beta 3
  • [26/11/2004] Lançamento da Versão 2.0 beta 2
  • [10/12/2003] Lançamento da Versão 2.0 beta

A Versão 2.0.0 introduziu o novo driver ADO e corrigiu alguns erros relativos ao driver ODBC.

A Versão 2.0 beta 3 corrigiu apenas alguns pequenos erros.

A Versão 2.0 beta 2 corrigiu alguns erros e introduziu os novos drivers SQLite and JDBC. Também acompanha a proposta de pacotes para Lua 5.1. Veja mais detalhes em Instalação.

A Versão 2.0 apresenta algumas modificações no design e aprimoramentos na implementação:

  • Novo método de busca: mais eficiente e mais flexível;
  • Novo método de setautocommit;
  • Compatível com Lua 5.0 e 5.1;
  • Carregável dinamicamente ou estaticamente;
  • Novos drivers para bancos de dados Oracle e MySQL.

Créditos

LuaSQL 2.0

A Versão 2.0 foi redesenhada por Roberto Ierusalimschy, André Carregal e Tomás Guisasola como parte do Projeto Kepler. A implementação é compatível com Lua 5.0 e foi codificada por Tomás Guisasola, Eduardo Quintão, Thiago Ponte, Fabio Mascarenhas, Danilo Tuler, com inestimáveis contribuições de Michael Roth, Tiago Dionisio e Leonardo Godinho.

LuaSQL 1.0

LuaSQL foi projetado por Pedro Miller Rabinovitch e Roberto Ierusalimschy. A primeira implementação era compatível com Lua 4.0a. Muitas modificações foram feitas, mas não distribuídas, por Diego Nehab (ODBC), Carlos Cassino, Tomás Guisasola and Eduardo Quintão (PostgreSQL).

O desenvolvimento de LuaSQL foi patrocinado pela Fábrica Digital.
